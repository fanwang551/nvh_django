**数据模型背景 (Context)**：
系统包含以下核心模型：
1.  `VehicleModel`: 车型基础信息（含 `vehicle_model_name`, `suspension_type`, `drive_type` 等）。
2.  `AirtightnessTest`: 关联车型，核心字段 `uncontrolled_leakage` (整车不可控泄漏量)。
3.  `VehicleSoundInsulationData`: 关联车型，核心字段 `sound_insulation_performance` (隔声性能)。
4.  `AcousticTestData`: 关联车型与工况(`condition_point`)，核心字段 `rms_value`。
5.  `SampleInfo`: 样品信息，part_name, odor_mean, odor_static_front, odor_dynamic_front, odor_static_rear, odor_dynamic_rear。


**第一行：核心数据概览 (KPI Cards)**
*   **布局**：4个等宽卡片。
*   **内容**：
    1.  **总试验数**：统计所有测试表（气密性+隔声+NTF+声学+VOC）的记录总和。
    2.  **年度试验数**：筛选本年度的上述试验总和。
    3.  **车型库**：`VehicleModel` 表的总记录数。
    4.  **样品数**：`SampleInfo` 表的总记录数。
*   **优化点**：每个卡片需包含一个图标。

**第二行：宏观趋势分析**
*   **布局**：左侧 70% 宽度，右侧 30% 宽度。
*   **左侧（折线图）**：**总试验数据月度趋势**。
    *   X轴：最近36个月（根据test_date等日期数据）。
    *   Y轴：试验数量。
*   **右侧（条形图）**：**各类试验数据量分布**。
    *   Y轴：试验类型（气密性、NTF、VOC、声学（整车隔声，区域隔声，整车混响时间））。
    *   X轴：数量。
    *   样式：使用横向柱状图，不同类型使用不同颜色区分。

**第三行：最新车型性能看板 (Rolling Table)**
*   **组件**：一个自动滚动的表格（或跑马灯效果的列表）。
*   **数据源**：`VehicleModel` 左连接 `AirtightnessTest` 和 `VehicleSoundInsulationData`。
*   **列定义**：
    1.  **车型名称** (`vehicle_model_name`)
    2.  **生产年份** (`production_year`)
    3.  **驱动/悬挂** (组合显示，例："EV / 多连杆")
    4.  **配置** (`configuration`，超出显示省略号)
    5.  **气密性 (SCFM)**：显示 `uncontrolled_leakage`。
    6.  **隔声性能 (dB)**：显示 `sound_insulation_performance`。
*   **交互**：鼠标悬停时停止滚动。

**第四行：专项测试深钻 (Carousel Charts)**
*   **布局**：左侧 50%，右侧 50%。
*   **左侧（雷达图 - 轮播）**：**各工况噪声分析**。
    *   **数据**：查询 `AcousticTestData`。
    *   **维度**：雷达图的轴为工况测点 ID [1, 5, 7, 9, 11, 13]。请将 ID 映射为工况名（ConditionMeasurePoint表）（如：1=idle AC on, 5=鹅卵石路60kph, 7=光滑沥青路60kph, 9=光滑沥青路80kph, 11=光滑沥青路100kph, 13=光滑沥青路120kph）。
    *   **数值**：`rms_value`。
    *   **动画**：每隔 5 秒切换展示最近录入的 3 个不同车型的数据。建议在图例中显示当前展示的车型名称。
*   **右侧（柱状图 - 轮播）**：**整车气味主观评价**。
    *   **数据**：查询 `SampleInfo` 中 `part_name` 包含"整车"的记录。
    *   **X轴**：样品为part_name="整车"的静态前排 动态前排 静态后排 动态后排 均值 。
    *   **Y轴**：气味值 (`odor_mean`等共5个值)，范围 1-10。
    *   **动画**：轮流展示最近的 5 个整车样品数据。
